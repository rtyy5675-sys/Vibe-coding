# 用户后续待确认事项

这些事项不需要现在全部提供。只有进入对应阶段时，才需要用户确认或补充。

## 1. 业务场景

需要时间：扩展 case 或进入真实模型评估前。

待确认：

- 优先评估的业务场景。
- 每个场景的关键失败模式。
- 哪些错误属于 Critical，必须单独统计。

## 2. 语言

需要时间：确定样本集前。

待确认：

- 主要语言。
- 是否包含多语言或混合语言。
- 是否需要方言、口语、省略表达或噪音场景。

## 3. 候选模型

需要时间：从 mock 模式切换到 real_model 模式前。

待确认：

- STT 候选模型。
- TTS 候选模型。
- LLM 候选模型。
- 每个模型的 provider、版本和调用方式。

## 4. API key

需要时间：执行真实模型评估前。

待确认：

- 是否提供 API key。
- API key 的使用边界。
- 是否允许记录 provider、模型版本和调用参数。
- 是否有费用或调用量限制。

## 5. 真实或脱敏数据

需要时间：v0.2 真实或半真实评估前。

待确认：

- 是否可以使用真实业务数据。
- 是否必须先脱敏。
- 脱敏规则由谁确认。
- 数据是否可以进入本地 `runs/` 和 `reports/`。
- 哪些字段不得落盘。

## 6. TTS voice_id

需要时间：执行 TTS 真实模型评估前。

待确认：

- 使用哪个 `voice_id`。
- 是否固定 voice，还是比较多个 voice。
- 是否需要按场景区分 voice。
- 是否有语速、情绪或音色配置要求。

## 7. 人工评分人员

需要时间：开始 MOS、开放答案或高风险 case 复核前。

待确认：

- 人工评分人员名单。
- 每类 case 的评分责任人。
- 是否需要双人评分。
- 分歧由谁仲裁。
- 人工评分结果是否可以写入报告。
