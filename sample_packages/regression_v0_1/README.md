# 平台回归测试包 v0.1

这是一组可直接通过首页上传的合成 fixture 用例。它不含生产数据、个人信息或真实模型结论，用于确认平台的材料校验、后台执行、评分、待复核、失败归档和报告生成是否正常。

每一类任务分别上传同名的 `*_cases.jsonl` 与 `*_outputs.jsonl`，模型类型应与文件名前缀一致。

| 类型 | 覆盖重点 | 预期任务状态 |
| --- | --- | --- |
| STT | 中英文、数字/实体、别名字段、实体缺失、缺失输出 | `partially_completed`；`STT-REG-006` 为失败样本 |
| TTS | 中英文、多实体、人工复核标记、实体缺失、无实体文本 | `completed`；结果中包含待复核和 Major 项 |
| LLM | 8 种本地断言、任一禁止短语、空禁止词、非 JSON、未知断言 | `completed`；失败断言保留为 Major 项，不中断任务 |

说明：本包刻意保留若干“评分失败”用例，以验证报告和人工复核链路；它们不是上传校验失败。STT 的一个 case 刻意缺少 output，用来验证任务级失败与重试。
